# 会议录音转写工具 — 产品需求文档（PRD）

> 本地离线会议录音 + 声纹分割 + 语音转写工具

---

## 一、产品概述

### 1.1 产品定位

面向个人用户的本地离线会议录音转写工具，支持 Windows 和 macOS 双平台。所有数据和处理均在本地完成，无需网络。

### 1.2 核心价值

- **隐私优先**：录音和转写完全本地化，不上传任何数据
- **低门槛**：一键录音，AI 自动区分说话人，自动转写
- **可追溯**：历史记录全文搜索，随时回放和导出

### 1.3 目标用户画像

- 需要记录会议内容的职场人士
- 参加多方通话后需要整理记录的用户
- 隐私敏感度高，拒绝云端处理的用户
- 使用低配电脑的个人用户（Mac M1 16GB / Windows 集显均可运行）

## 二、功能需求

### 2.1 会议录音

**功能列表：**
- 一键开始 / 暂停 / 停止录音
- 实时波形可视化（麦克风音量实时显示）
- 录音时长实时显示
- 录音中实时显示检测到的说话人数量
- 实时字幕显示（功能开关可控制）
- 录音质量提示（麦克风音量过大 / 过小提示）
- 意外中断后自动恢复录音

**技术规格：**
- 采样率：16kHz（转写标准）
- 格式：WAV
- 通道：单声道

---

### 2.2 音频导入

**功能列表：**
- 支持导入格式：MP3, M4A, WAV, FLAC, OGG, OPUS, AAC, WMA
- 拖拽文件或点击选择（支持多选）
- 批量导入队列，显示每个文件的处理进度
- 自动命名：使用文件名或创建时间
- 导入前预览：显示时长、声道数、采样率
- 跳过已导入文件（按文件名 hash 识别）

**技术规格：**
- 使用 FFmpeg 将所有格式转换为 WAV 16kHz 单声道后处理

---

### 2.3 说话人分割（Speaker Diarization）

**功能列表：**
- 自动检测当前发言者
- 自动识别有几个人在说话
- 按人分组显示转写结果
- 支持最多 8 人同时开会
- 说话人标签命名（Speaker 1, Speaker 2 ...）
- 用户可手动重命名说话人
- 用户可为说话人分配颜色

**技术规格：**
- 使用 pyannote.audio 3.x
- 默认参数：min_speakers=2, max_speakers=8

---

### 2.4 语音转写

**功能列表：**
- 中文语音识别（中文优先，不支持英文会议）
- 自动标点（SenseVoice 自带）
- 语气词自动过滤（呃、啊、嗯等）
- 按人聚合转写结果
- 完整转写结果时间戳对齐

**技术规格：**
- 使用 FunAudioLLM / SenseVoice Small（0.27B 参数）
- 本地推理，无需网络

---

### 2.5 历史记录管理

**功能列表：**
- 列表展示：标题、时间、时长、说话人数量、处理状态
- 全局全文搜索（搜索转写内容）
- 按时间范围筛选（今天 / 本周 / 本月 / 自定义）
- 按说话人数量筛选
- 按标签筛选（支持多标签）
- 收藏 / 取消收藏
- 批量删除、批量导出
- 列表排序：按时间 / 按时长 / 按说话人数
- 会议标题可编辑
- 会议备注可添加

---

### 2.6 转写结果查看

**功能列表：**
- 音频播放：播放 / 暂停 / 进度拖拽 / 倍速（0.5x, 1x, 1.5x, 2x）
- 时间线同步高亮：播放到某句时，对应文字高亮并自动滚动
- 按说话人分组展示（可折叠 / 展开）
- 每个人单独查看
- 转写文本在线编辑（点击编辑）
- 说话人重命名
- 说话人颜色自定义
- 标签管理（添加 / 删除标签）
- 会议备注编辑
- 发言时长统计（各人发言时长占比）

---

### 2.7 导出功能

**功能列表：**
- TXT 导出：纯文本，按时间排序，含说话人标签
- JSON 导出：完整结构化数据（含时间戳、说话人、文本）
- Markdown 导出：可读性好，按人分组，含时间戳
- SRT 字幕导出：带时间轴的字幕格式
- 批量导出：选中多个会议，一键打包 ZIP
- 导出时可选是否包含音频文件

---

### 2.8 设置

**功能列表：**
- 麦克风设备选择（下拉选择系统麦克风）
- 语言设置：中文（默认）
- 实时字幕开关：on / off
- 实时字幕刷新间隔：1秒 / 2秒 / 3秒 / 5秒
- 导出默认格式：TXT / JSON / Markdown / SRT
- 音频质量：16kHz（转写用）/ 44.1kHz（高音质）
- 开机启动（可选）
- 系统托盘：最小化到托盘 / 关闭退出
- 模型管理：查看已下载模型、清除模型缓存
- 数据管理：清除所有历史记录、导出 / 导入数据库

## 三、非功能性需求

### 3.1 性能

| 场景 | 目标 |
|------|------|
| 实时字幕延迟 | < 3 秒 |
| 30 分钟会议转写（M1 Mac 16GB） | < 5 分钟 |
| 30 分钟会议转写（Windows 集显） | < 15 分钟 |
| 启动时间 | < 3 秒 |

### 3.2 兼容性

| 平台 | 最低配置 |
|------|----------|
| macOS | Apple Silicon（M1/M2/M3），16GB 统一内存 |
| Windows | Intel / AMD 集显，16GB RAM |

### 3.3 数据安全

- 所有数据存储在本地，不上传任何服务器
- 数据库存储在用户应用数据目录下
- 录音文件存储在用户应用数据目录下

## 五、技术架构

### 5.1 技术选型

| 层次 | 技术 |
|------|------|
| 桌面框架 | Electron 33 + electron-builder |
| 前端 | React 18 + TypeScript + Vite |
| 样式 | TailwindCSS |
| 状态管理 | Zustand |
| 音频录制 | Python sounddevice |
| 声纹分割 | pyannote.audio 3.x |
| 语音转写 | FunAudioLLM / SenseVoice Small |
| VAD | Silero-VAD |
| 本地数据库 | SQLite + better-sqlite3 |
| IPC | stdio JSON-RPC |
| 音频格式 | FFmpeg 静态二进制 |
| 国际化 | i18next（预留）|

### 5.2 项目结构

```
meeting-recorder/
├── electron/           # Electron 主进程
├── src/                # React 前端
├── python/             # Python 后端模块
├── resources/          # FFmpeg 等资源
├── package.json
└── electron-builder.yml
```

## 六、验收标准

| 功能 | 验收条件 |
|------|----------|
| 录音 | 录 10 分钟，播放正常，文件可被其他播放器打开 |
| 说话人分割 | 两人对话 10 分钟，能区分两个说话人，正确率 > 80% |
| 转写 | 中文普通话 10 分钟，转写文本可读，无明显错误 |
| 实时字幕 | 录音时说中文，3 秒内显示字幕 |
| 历史搜索 | 搜索关键词能精确找到含该词的会议 |
| 导出 MD | 用 Markdown 编辑器打开格式正确 |
| macOS 打包 | .dmg 安装后能正常运行 |
| Windows 打包 | .exe 安装后能正常运行 |
